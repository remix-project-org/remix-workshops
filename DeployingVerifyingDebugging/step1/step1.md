# Deploying, Verifying & Debugging

## Introduction

This tutorial covers the complete workflow for deploying, verifying, and debugging smart contracts in Remix IDE. You'll learn about deployment options, contract verification, frontend generation, and debugging tools.

## Topics Covered

### 1. Deploy & Run
Learn to use the Deploy & Run plugin including using the AtAddress feature, use of low level interactions and pinning contracts.

**Video**: [Deploy & Run](https://www.youtube.com/watch?v=f8DnHN0v4fw)

### 2. Verifying Contracts
Learn how to use the Contract Verification plugin to verify your smart contracts on block explorers like Etherscan.

**Video**: [Verifying Contracts](https://www.youtube.com/watch?v=fupWp7ONeKY)

### 3. Generate a Frontend
QuickDapp is a tool for quickly making a simple front end to a smart contract. A great tool for hackathons!

**Video**: [Generate a Frontend](https://www.youtube.com/watch?v=l5qVj1xLm8s)

### 4. Remix VM & Forking
The Remix VM is the local in-browser test blockchain. Learn about its features and how to fork and how to share the chain's state.

**Video**: [Remix VM & Forking](https://www.youtube.com/watch?v=oYkXApf36wk)

### 5. Transaction Recorder
Learn how to record and replay transactions for testing and debugging purposes.

**Video**: [Transaction Recorder](https://www.youtube.com/watch?v=GchvmIRSxUo)

## Next Steps

With these deployment and debugging skills, you'll be able to confidently deploy and verify your smart contracts on various networks.
